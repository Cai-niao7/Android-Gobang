# Android 五子棋游戏应用 - 需求文档

## 1. 项目概述

### 1.1 项目背景
基于 Android 平台开发一款五子棋游戏应用，提供人机对战和本地双人对战两种模式，支持用户注册登录、游戏记录统计、排行榜等功能。

### 1.2 项目目标
- 实现完整的五子棋游戏逻辑（落子、悔棋、胜负判定）
- 提供 AI 对战功能，支持不同难度级别
- 建立用户系统，实现用户数据持久化
- 提供游戏统计与排行榜功能

### 1.3 技术选型
| 技术 | 用途 |
|------|------|
| Java 11 | 开发语言 |
| Android SDK 34 | 基础框架 |
| Room 2.8.4 | 本地数据库 ORM |
| Material Design | UI 组件 |
| SoundPool | 短音效播放 |
| MediaPlayer | 背景音乐 |
| ExecutorService | 异步线程管理 |

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 用户注册
- **功能描述**：新用户通过账号密码注册
- **输入要求**：
  - 账号：1-16 位字符，不可重复
  - 密码：6-18 位字符，需二次确认
- **校验规则**：
  - 账号/密码非空校验
  - 密码一致性校验
  - 账号唯一性校验（查询数据库）
- **输出**：注册成功提示，1.5 秒后自动返回登录页

#### 2.1.2 用户登录
- **功能描述**：已注册用户通过账号密码登录
- **输入要求**：账号、密码
- **校验规则**：
  - 账号/密码非空校验
  - 账号存在性校验
  - 密码正确性校验
- **输出**：登录成功跳转主菜单，失败提示具体原因（账号未注册/密码错误）

#### 2.1.3 找回密码
- **功能描述**：输入已注册账号，显示对应密码
- **输入要求**：账号
- **输出**：显示密码或提示账号未注册

### 2.2 主菜单模块

#### 2.2.1 功能入口
- 人机对战入口
- 本地双人对战入口
- 排行榜入口
- 个人中心入口
- 音乐控制开关
- 退出游戏

#### 2.2.2 用户信息展示
- 显示当前登录用户头像（从数据库加载）
- 显示当前登录用户账号

### 2.3 人机对战模块

#### 2.3.1 游戏设置
- **难度选择**：简单 / 困难（游戏开始后不可更改）
- **执棋颜色**：玩家执黑 / 玩家执白（游戏开始后不可更改）
- **默认设置**：简单难度，玩家执黑

#### 2.3.2 游戏控制
- **暂停/继续**：暂停游戏并停止计时，继续后恢复
- **悔棋**：撤销玩家和 AI 各一步（棋子总数 > 10 时可用）
- **返回**：提供重新开始、返回主菜单选项

#### 2.3.3 游戏信息展示
- 黑棋步数 / 白棋步数
- 当前回合信息（玩家回合 / AI 回合）
- 游戏计时（MM:SS 格式）

#### 2.3.4 游戏结束处理
- 显示胜负结果
- 显示游戏时间、总步数统计
- 自动保存游戏记录到数据库
- 提供退出 / 继续查看选项

#### 2.3.5 AI 行为
- AI 落子延迟 800ms（模拟思考时间）
- AI 先手时延迟 1000ms 自动落子
- 应用进入后台时自动暂停游戏

### 2.4 本地双人对战模块

#### 2.4.1 游戏控制
- **暂停/继续**：暂停游戏，禁用悔棋按钮
- **悔棋**：撤销一步棋
- **返回**：直接返回上一页面

#### 2.4.2 游戏信息展示
- 黑棋步数 / 白棋步数
- 当前回合信息（黑棋回合 / 白棋回合）
- 根据当前玩家切换回合信息背景色

#### 2.4.3 游戏结束处理
- 显示胜负结果
- 提供重新开始 / 继续 / 退出选项

### 2.5 个人中心模块

#### 2.5.1 用户信息展示
- 显示用户头像
- 显示用户名
- 显示简单模式统计数据：胜率、总对局、胜利局数
- 显示困难模式统计数据：胜率、总对局、胜利局数

#### 2.5.2 用户信息管理
- **修改用户名**：
  - 输入新用户名（1-16 位字符）
  - 校验用户名唯一性
  - 更新数据库并同步全局登录状态
- **更换头像**：
  - 从 10 个头像中随机选择
  - 更新数据库并刷新显示
- **切换账户**：
  - 确认对话框
  - 清除登录状态，返回登录页

### 2.6 排行榜模块

#### 2.6.1 排行榜展示
- 支持简单模式 / 困难模式切换
- 按胜率降序排列（胜率相同按总场次降序）
- 显示排名、头像、账号、胜率
- 显示当前用户排名和胜率

#### 2.6.2 玩家详情
- 点击列表项显示玩家详细信息
- 包含：玩家账号、排名、胜率、总场次、胜场

### 2.7 音效模块

#### 2.7.1 背景音乐
- 循环播放背景音乐
- 支持暂停 / 恢复
- 支持音量控制
- 支持开关切换

#### 2.7.2 游戏音效
- 按键音效
- 落子音效
- 胜利 / 失败音效
- 支持开关切换

## 3. 非功能需求

### 3.1 性能需求
- 数据库操作在后台线程执行，不阻塞主线程
- AI 落子延迟模拟思考，避免瞬时响应
- 棋盘绘制使用自定义 View，保证流畅渲染

### 3.2 兼容性需求
- 最低支持 Android 8.0（API 26）
- 目标 SDK 34（Android 14）
- 适配不同分辨率设备的棋盘自适应布局

### 3.3 数据一致性需求
- 游戏记录与用户通过外键关联，支持级联删除
- 用户修改用户名后同步更新全局登录状态
- 游戏结束自动保存记录，确保数据不丢失

### 3.4 用户体验需求
- 所有按钮操作伴随音效反馈
- 游戏状态变化实时更新 UI
- 异常操作提供明确提示（如悔棋限制、难度不可更改等）

## 4. 技术架构

### 4.1 架构模式
采用 **MVC 架构**：

```
┌─────────────────────────────────────────────────┐
│                    View 层                       │
│  boardView（自定义棋盘View） + Activity 布局 XML  │
├─────────────────────────────────────────────────┤
│                 Controller 层                    │
│  MainActivity / PvcGamingActivity / ...          │
├─────────────────────────────────────────────────┤
│                  Model 层                        │
│  GameLogic / GameAi / User / GameRecord / Room   │
└─────────────────────────────────────────────────┘
```

### 4.2 模块设计

#### 4.2.1 游戏逻辑模块（GameLogic）
- 管理棋盘状态（15×15 二维数组）
- 处理落子、悔棋、重置操作
- 胜负判定（四方向五子连珠检测）
- 通过 `GameLogicListener` 接口通知 UI 层

#### 4.2.2 AI 引擎模块（GameAi）
- 简单难度：60% 困难算法 + 40% 玩家周围随机落子
- 困难难度：评分算法（进攻权重 10，防守权重 8）
- 棋型评分：活四(1000)、活三(100)、冲三(50)、活二(10)、冲二(5)、单子(1)
- 中心位置加分（曼哈顿距离）

#### 4.2.3 棋盘视图模块（boardView）
- 绘制棋盘网格和棋子
- 处理触摸事件，转换屏幕坐标为棋盘坐标
- AI 回合屏蔽玩家触摸
- 通过 `GameEventListener` 接口传递事件给 Activity

#### 4.2.4 全局管理模块（GameApp）
- 初始化 Room 数据库
- 管理全局线程池（ExecutorService）
- 管理音频管理器（AudioManager）
- 管理当前登录账号状态

### 4.3 异步处理设计
- 数据库操作：`ExecutorService` 线程池执行
- UI 更新：`Handler` / `runOnUiThread` 切换主线程
- AI 落子：`postDelayed` 延迟执行

## 5. 数据库设计

### 5.1 User 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | 主键，自增 | 用户唯一标识 |
| user_account | TEXT | 唯一 | 用户账号 |
| user_password | TEXT | - | 用户密码 |
| avatar_id | INTEGER | 默认 0 | 头像 ID（1-10） |

### 5.2 GameRecord 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| record_id | INTEGER | 主键，自增 | 记录唯一标识 |
| user_id | INTEGER | 外键，级联删除 | 关联 User.id |
| is_easy_mode | INTEGER | - | 是否为简单模式（1:是，0:否） |
| is_win | INTEGER | - | 是否获胜（1:是，0:否） |
| record_time | INTEGER | - | 记录时间戳 |

### 5.3 表关系
```
User (1) ──< GameRecord (N)
```
- 一个用户可以有多条游戏记录
- 删除用户时自动删除其所有游戏记录（CASCADE）

## 7. 测试策略

### 7.1 测试用例设计方法
- **等价类划分**：有效/无效输入分类（如账号长度、密码格式）
- **边界值分析**：临界值测试（如密码 6 位/18 位、棋盘边缘落子）
- **场景法**：完整业务流程覆盖（注册→登录→对战→排行）

### 7.2 测试类型

#### 7.2.1 单元测试
- 游戏逻辑测试：
  - 胜负判定（水平/垂直/对角线五子连珠）
  - 悔棋回滚（栈结构正确性）
  - 棋盘重置（状态清零）
- AI 算法测试：
  - 评分算法正确性
  - 难度切换功能

#### 7.2.2 集成测试
- Room 数据库测试：
  - CRUD 操作正确性
  - 外键级联删除
  - 用户注册查重
- 线程通信测试：
  - 后台线程→主线程 UI 更新
  - 异步操作顺序性

#### 7.2.3 系统测试
- 端到端业务流程测试：
  - 注册→登录→人机对战→保存记录→查看排行榜
  - 注册→登录→本地对战→个人中心查看统计
- 页面跳转与生命周期测试

#### 7.2.4 异常测试
- 数据库操作异常处理
- 线程阻塞场景
- AI 决策超时场景
- 空指针/空值处理

#### 7.2.5 兼容性测试
- 不同分辨率设备棋盘自适应
- 不同 Android 版本兼容性

### 7.3 缺陷管理流程
```
发现缺陷 → 记录缺陷 → 分析根因 → 修复缺陷 → 回归验证 → 关闭缺陷
```
